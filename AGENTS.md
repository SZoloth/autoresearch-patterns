# AGENTS.md

## What this repo is

A configurable tool for running autonomous iterative experiments. Users write a `lab.yaml`, run `init.sh`, and get a generated `program.md` that any AI agent can follow to optimize a metric.

The key insight: `program.md` is the product. Everything else exists to generate a good one from user config.

## Architecture (v0.7)

```
lab.yaml → init.sh → (parse_config.py + render.py) → program.md + benchmark.sh
                   → graph.py → .autoresearch/graph.json (experiment tree)
                   → scratchpad.py → autoresearch.md (auto-generated state)
```

### Experiment tree (v0.7+)

Experiments form a tree, not a line. Each experiment is a node with parent/children pointers, stored in `.autoresearch/graph.json`. Failed experiments are preserved as data (not erased by git reset).

Key commands:
- `autoresearch record <status> <value> [--type T] [--description D]` — record an experiment
- `autoresearch fork <commit>` — branch from any committed node
- `autoresearch tree` — show the experiment tree
- `autoresearch recover` — reconcile graph with git history

## Structure

- `bin/autoresearch` — CLI dispatcher (symlinked to PATH by install.sh)
- `install.sh` — curl-installable installer (clones repo + symlinks)
- `init.sh` — core orchestrator: parse config → render templates → create branch → init graph
- `lib/parse_config.py` — parses lab.yaml (stdlib only, tries PyYAML then falls back to simple parser)
- `lib/render.py` — renders templates with `{{var}}` substitution, `{{#each}}`, `{{#if}}`
- `lib/interactive.py` — interactive questionnaire, generates lab.yaml from user answers
- `lib/graph.py` — experiment tree operations (load/save/add_node/frontier/negative knowledge)
- `lib/experiment.py` — per-experiment storage (result.json, diff.patch, trace.log)
- `lib/scratchpad.py` — auto-generated state summary (replaces manual autoresearch.md)
- `lib/record.py` — record experiment results into the graph
- `lib/fork_node.py` — fork from any committed node
- `lib/tree_view.py` — display experiment tree
- `lib/recover.py` — reconcile graph.json with git history
- `lib/migrate.py` — migrate v0.6 sessions to v0.7 tree format
- `lib/status.py` — session status display (tree-aware with fallback to results.tsv)
- `lib/learn.py` — cross-session skill extraction
- `templates/` — Handlebars-style templates for generated files
- `examples/` — pre-built lab.yaml configs for common optimization targets
- `reference/` — original source files from karpathy/autoresearch and pi-autoresearch

## Guidelines

- Python code uses stdlib only — no pip dependencies, ever
- Templates use `{{var}}` syntax with dot notation (`{{metric.name}}`)
- Generated `program.md` must be completely self-contained — no imports, no references to this repo
- The three invariants: immutable evaluation, mutable artifact, deterministic keep/discard
- `.autoresearch/` is .gitignored — working state, not committed to branches
- `results.tsv` stays append-only for backwards compat (dual-written by `record`)
- `autoresearch.md` is now auto-generated by scratchpad.py (not agent-maintained)

## Testing changes

After modifying templates or lib code, verify by running `init.sh` with each example yaml in a test git repo and checking the generated files make sense. Test the full loop: init → test → manual commit → record keep → record discard → fork → tree → status.
